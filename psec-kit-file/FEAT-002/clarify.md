# Clarification & Decisions – FEAT-002

## 1. Executive Summary
ميزة التسجيل وتسجيل الدخول تمثل بوابة الدخول الأساسية للمنصة، وتهدف إلى تمكين المستخدمين من إنشاء حساب شخصي وتسجيل الدخول باستخدام البريد الإلكتروني فقط، مع توفير حماية أساسية للحسابات دون تعقيد يناسب بيئة MVP وسياق الاستخدام في سوريا.

## 2. Scope Confirmation
- Confirmed In-Scope:
  - تسجيل مستخدم جديد باستخدام البريد الإلكتروني وكلمة مرور.
  - تسجيل الدخول باستخدام البريد الإلكتروني وكلمة المرور.
  - تشفير كلمات المرور وحمايتها.
  - جلسة مستخدم أساسية (Authentication Session).
- Confirmed Out-of-Scope:
  - تسجيل باستخدام شبكات اجتماعية.
  - تفعيل البريد الإلكتروني أو استعادة كلمة المرور عبر الإيميل.
  - أدوار وصلاحيات متقدمة.
  - المصادقة الثنائية (2FA).

## 3. Ambiguity Resolution (Decisions Catalog)

| Decision ID | Topic | Uncertainty | Decision Taken | Rationale |
|-------------|-------|-------------|----------------|-----------|
| DEC-FEAT-002-001 | Auth Method | هل يسمح بطرق تسجيل متعددة؟ | البريد الإلكتروني فقط | تقليل التعقيد في MVP |
| DEC-FEAT-002-002 | Password Policy | مستوى تعقيد كلمة المرور | حد أدنى 8 أحرف | توازن بين الأمان وسهولة الاستخدام |

## 4. Key Business Rules Identified
- لا يمكن إنشاء حساب دون بريد إلكتروني فريد.
- البريد الإلكتروني هو المعرّف الأساسي للمستخدم.
- كلمة المرور مخزنة مشفرة (Hash).
- المستخدم غير المسجل يمكنه استعراض الفعاليات فقط.

## 5. Implementation Recommendations
- الاعتماد على ABP Identity Module دون تخصيصات كبيرة.
- توفير نماذج تسجيل بسيطة تدعم اللغة العربية بالكامل.
- معالجة أخطاء تسجيل الدخول برسائل واضحة وغير تقنية.
