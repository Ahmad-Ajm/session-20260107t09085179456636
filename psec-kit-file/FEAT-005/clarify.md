# Clarification & Decisions – FEAT-005

## 1. Executive Summary
- هذه الميزة تهدف إلى إضافة روابط ثابتة وواضحة لصفحات سياسة الخصوصية وشروط الاستخدام ضمن واجهة المستخدم، دون بناء نظام قانوني متكامل، وذلك لتغطية الحد الأدنى من المتطلبات القانونية والتجهيزية للمنصة في مرحلة MVP.

## 2. Scope Confirmation
- Confirmed In-Scope:
  - إضافة رابط "سياسة الخصوصية".
  - إضافة رابط "الشروط والأحكام".
  - روابط نصية فقط تقود إلى صفحات ثابتة.
- Confirmed Out-of-Scope:
  - نظام موافقة قانونية متقدم.
  - تتبع قبول المستخدم للشروط.
  - محتوى قانوني مخصص أو ديناميكي.

## 3. Ambiguity Resolution (Decisions Catalog)

| Decision ID | Topic | Uncertainty | Decision Taken | Rationale |
|-------------|-------|-------------|----------------|-----------|
| DEC-FEAT-005-001 | Content Source | هل المحتوى ديناميكي أو ثابت؟ | استخدام صفحات ثابتة (Static Pages) | أبسط وأسرع للتنفيذ في MVP |
| DEC-FEAT-005-002 | Language | هل تدعم لغة واحدة أم اكثر؟ | العربية فقط في MVP | المنصة موجهة للجمهور المحلي |
| DEC-FEAT-005-003 | Backend | هل تحتاج API خاص؟ | لا حاجة لـ API مستقل | المحتوى ثابت ويمكن تحميله من الواجهة |
| DEC-FEAT-005-004 | UI Location | مكان عرض الروابط غير محدد | وضع الروابط في الفوتر | ممارسة شائعة ومفهومة للمستخدم |

## 4. Key Business Rules Identified
- يجب أن تكون الروابط مرئية من جميع الصفحات العامة.
- محتوى الصفحات للعرض فقط بدون تفاعل.

## 5. Implementation Recommendations
- إنشاء مكونات Angular بسيطة لصفحات السياسة والشروط.
- عدم ربط الميزة بمنطق Backend لتقليل التعقيد.
